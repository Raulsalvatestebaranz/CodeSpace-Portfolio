How containers work:
.container         → Fixed width at breakpoints
.container-{bp}    → 100% width until that breakpoint, then fixed
.container-fluid   → Always 100% width

Responsive containers:
.container-sm      → 100% until ≥576px
.container-md      → 100% until ≥768px
.container-lg      → 100% until ≥992px
.container-xl      → 100% until ≥1200px
.container-xxl     → 100% until ≥1400px

Fluid container:
.container-fluid   → Always full width

How the grid works:
.container       → Wraps content, centers it horizontally
.row             → Wraps columns and manages horizontal spacing (gutters)
.col             → Auto width column (equal widths unless specified)
.col-{n}         → Column spans n of 12 total
.col-{bp}-{n}    → Responsive column for each breakpoint

Responsive column classes:
.col             → Equal width across all breakpoints
.col-sm-*        → Starts from ≥576px
.col-md-*        → Starts from ≥768px
.col-lg-*        → Starts from ≥992px
.col-xl-*        → Starts from ≥1200px
.col-xxl-*       → Starts from ≥1400px

Auto-layout columns:
.col             → Equal width columns using flexbox
.col-6           → Takes 6 of 12 columns
.col-auto        → Width automatically fits content

Nesting columns:
.col > .row      → Create nested grids
  └─ .row
     ├─ .col-sm-4
     └─ .col-sm-8

Row column utilities:
.row-cols-2      → Two equal columns
.row-cols-3      → Three equal columns
.row-cols-auto   → Columns sized to content width

Gutters (spacing between columns):
.g-0 to .g-5     → Set gutter spacing (0 = no gap)
.gx-*            → Horizontal gutters only
.gy-*            → Vertical gutters only
                    

How columns work:
Columns build on the grid system using flexbox.
.container → .row → .col is the hierarchy for layout.
Predefined classes span 12 columns per row.

Vertical alignment (row-based):
.align-items-start     → Align columns to top
.align-items-center    → Align columns to middle
.align-items-end       → Align columns to bottom

Vertical alignment (col-based):
.align-self-start      → Align this column to top
.align-self-center     → Align this column to middle
.align-self-end        → Align this column to bottom

Horizontal alignment (row-based):
.justify-content-start     → Align left
.justify-content-center    → Center horizontally
.justify-content-end       → Align right
.justify-content-between   → Space between
.justify-content-around    → Space around
.justify-content-evenly    → Evenly spaced

Column wrapping:
If total columns > 12, next columns wrap to new line
Use <div class="w-100"></div> to force manual line break

Reordering with flexbox:
.order-1 to .order-5         → Change column order
.order-first / .order-last   → Move to first or last

Offsetting columns:
.offset-md-4    → Push column right by 4 columns
.offset-sm-2    → Push right from sm breakpoint

Margin-based spacing:
.ms-auto        → Push column to the right
.me-auto        → Push column to the left

Standalone column widths:
.col-3          → Width: 25%
.col-sm-9       → Width: 75% from ≥576px

Floating media & text:
.clearfix + .float-md-end + .col-md-6 → Float image with responsive text wrapping
                    

How gutters work:
Gutters = spacing (padding) between columns
Default: 1.5rem (24px)
Set by horizontal padding on cols + negative margin on row

Horizontal gutters (.gx-*):
gx-0 to gx-5 → Adjust horizontal spacing between columns
.row.gx-5 → Large gap horizontally

Vertical gutters (.gy-*):
gy-0 to gy-5 → Adjust vertical spacing (especially when wrapping)
.row.gy-5 → Vertical spacing between rows

Both directions (.g-*):
g-0 to g-5 → Applies both horizontal and vertical spacing
.row.g-2 → Even gutter spacing both ways

Responsive gutters:
g-lg-3 → Gutter spacing at large breakpoint
gx-sm-4 → Horizontal only for small and up

No gutters:
.g-0 → Removes all gutter spacing
Useful for edge-to-edge designs
Pair with .mx-0 to eliminate row overflow

Overflow fix:
Wrap .row in .overflow-hidden to clip overflow

Row columns + gutters:
.row-cols-2.g-2         → 2 columns per row with spacing
.row-cols-lg-5.g-lg-3   → Responsive row-cols with gutter spacing

How it works:
CSS Grid is opt-in. Enable it by setting:
$enable-grid-classes: false
$enable-cssgrid: true
Replace .row with .grid, and use .g-col-* for columns.

Key differences:
• Uses grid-column instead of width
• Gaps (gap, row-gap, column-gap) replace gutters
• No negative margins; margin utilities don't affect gaps

JavaScript to Activate Custom Validation:

(() => {
  'use strict';
  const forms = document.querySelectorAll('.needs-validation');
  Array.from(forms).forEach(form => {
    form.addEventListener('submit', event => {
      if (!form.checkValidity()) {
        event.preventDefault();
        event.stopPropagation();
      }
      form.classList.add('was-validated');
    }, false);
  });
})();

Guiding principles:
• Mobile-first and responsive components.
• Use of base classes (e.g., .btn) extended with modifiers (e.g., .btn-primary).
• Common z-index scale: 0 (default), 1 (:hover), 2 (:active), 3 (:focus).
• Overlay components (dropdowns, modals, etc.) start at z-index 1000+.
• Preference for HTML & CSS over JavaScript.
• Utility-first approach to reduce custom CSS.
• Minimal HTML structure requirements for flexible implementation.

Component Strategy:
• Base class defines common styles (e.g., .btn).
• Modifier classes add variations (e.g., .btn-success, .btn-lg).
• Avoid excessive specificity and strict children selectors.

Utilities:
• Encourage immutability: .d-block → display: block;
• Great for speeding up development and reducing CSS bloat.
• Custom utilities should be added to $utilities Sass map.

Coding Standards:
• Follow Bootstrap’s Code Guide (@mdo).
• Stylelint for CSS/Sass formatting.
• vnu-jar for semantic and valid HTML.
